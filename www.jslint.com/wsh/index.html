JSLint and WSH (Windows Script Host)

Windows Script Host is a scripting feature for Microsoft Windows. It can be used to run JSLint from the command line on Windows systems.

The file jslint.js can be run by WSH. It will read StdIn (Standard Input). If there are no problems, it terminates normally. If there is a problem, it outputs a message to StdErr (Standard Error) and exits with error 1. The WSH version of JSLint does not produce a Function Report.

One way to run JSLint is with this command:

C:\> cscript jslint.js <myprogram.js

It runs cscript (the command-line version of WSH) which loads and runs jslint.js, which will read the file myprogram.js from the Standard Input.
